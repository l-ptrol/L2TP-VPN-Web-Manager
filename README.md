# L2TP VPN Web Manager для Keenetic

Веб-менеджер для управления L2TP/L2TP-IPsec VPN-соединениями на роутерах Keenetic с установленной средой Entware. Предоставляет удобный веб-интерфейс и полностью автоматизирует настройку xl2tpd, strongSwan (IPsec) и маршрутизации — всё в одном бинарном файле без внешних зависимостей.

---

## Зачем это нужно

### Проблема: Конфликт встроенного L2TP/IPsec в сетях с «белым» IP

Основная проблема роутеров Keenetic заключается в следующем:

1. **Невозможность работы встроенного L2TP/IPsec клиента на «белом» IP**:
   Если провайдер предоставляет роутеру публичный (выделенный, статический или динамический «белый») IP-адрес на WAN-интерфейсе, встроенный клиент L2TP/IPsec в KeeneticOS **не может установить соединение** с удаленным сервером. Происходит внутренний конфликт портов 500/4500 и политик безопасности с системными службами IPsec самой KeeneticOS (которые зарезервированы под встроенный VPN-сервер или Site-to-Site соединения).

2. **Необходимость сложной ручной настройки в Entware**:
   Поскольку стандартными средствами прошивки подключить L2TP/IPsec на «белом» IP невозможно, пользователям приходится ставить сторонние пакеты из Entware (`strongSwan` + `xl2tpd` + `pppd`). Настройка этой связки вручную через консоль крайне сложна: нужно правильно настроить параметры шифрования, авторизации, согласовать запуск демонов и прописать правила NAT. Любое обновление Entware может легко сломать конфигурацию.

---

### Решение: L2TP VPN Web Manager

Наш менеджер полностью автоматизирует обход этого ограничения прошивки, превращая ручную настройку Entware в простое и надежное решение:

1. **Бесшовный запуск L2TP/IPsec на «белом» IP**:
   Утилита генерирует изолированные конфигурации для `xl2tpd` и `strongSwan`, обходит конфликты портов с системными службами KeeneticOS и гарантирует успешный подъем L2TP/IPsec туннеля на внешних интерфейсах с любым типом IP (включая статические белые IP).

2. **Встроенный SOCKS5-прокси на loopback**:
   Для сценариев, когда трафик нужно пустить через туннель выборочно (например, только для определенных сайтов в браузере через расширения вроде SwitchyOmega), менеджер запускает защищенный SOCKS5-прокси на `127.0.0.1`. Исходящий трафик прокси жестко привязывается к интерфейсу туннеля (например, `ppp10`) с помощью системного вызова `SO_BINDTODEVICE`.

3. **Интеллектуальный Multi-WAN Failover**:
   Возможность указать приоритетный список внешних интерфейсов роутера (например, `ISP, UsbLte0`). При переключении основного интернета на резервный менеджер сам пересоздает туннель на новом интерфейсе.

4. **Автозапуск и Вочдог**:
   Сторожевой таймер опрашивает статус туннеля каждые 5 секунд и автоматически перезапускает его в случае падения, гарантируя постоянную доступность. Выбранный профиль можно назначить для автозапуска при загрузке роутера.

## Возможности

- **Мультипрофили** — несколько VPN-конфигураций с быстрым переключением
- **L2TP + IPsec** — полная поддержка strongSwan (swanctl) с автоматическим поднятием IPsec SA перед L2TP
- **Без IPsec** — также поддерживается чистый L2TP для серверов, не требующих шифрования
- **SOCKS5-прокси** — встроенный защищенный прокси-сервер на `127.0.0.1:<port>`, туннелирующий весь трафик через VPN
- **Автовыбор интерфейса** — указание нескольких WAN-интерфейсов с приоритетом; автоматическое переключение при недоступности основного
- **Watchdog** — мониторинг туннеля каждые 5 секунд, автоматический перезапуск при обрыве
- **Автозапуск** — запуск выбранного профиля при старте роутера (с задержкой 10 сек для инициализации сети)
- **Детальные логи** — агрегация логов из syslog, swanctl.log, ppp-vpn.log, xl2tpd.log и Go-бэкенда с хронологической сортировкой
- **Авторизация PPP** — выбор типа аутентификации: Auto, PAP, CHAP, MS-CHAP, MS-CHAP v2
- **Кроссплатформенная сборка** — бинарники под mipsle (основная масса Keenetic), arm, arm64, amd64
- **Отзывчивый UI** — современный веб-интерфейс, полностью адаптированный под мобильные устройства и планшеты

## Установка

### Требования

- Роутер **Keenetic** с установленной средой **Entware**
- Пакеты: `strongswan-default`, `strongswan-swanctl`, `xl2tpd`, `ip-full`, `iptables`, `ppp-mod-pppol2tp`
- SSH-доступ к роутеру (root)

### Быстрая установка (через SSH)

Подключитесь к роутеру по SSH (например, `ssh root@192.168.1.1`) и выполните команду (вы можете заменить `8081` в конце команды на любой другой свободный порт):

```bash
opkg update && opkg install wget-ssl && wget -qO- https://raw.githubusercontent.com/l-ptrol/L2TP-VPN-Web-Manager/main/l2tp-setup.sh | sh -s 8081
```

Скрипт полностью автоматизирован и выполняет следующие шаги:
1. **Проверка окружения**: Убедится в наличии Entware.
2. **Установка зависимостей**: Автоматически установит из репозитория Entware все необходимые пакеты (`strongswan-default`, `strongswan-swanctl`, `strongswan-mod-openssl`, `xl2tpd`, `ip-full`, `iptables`, `ppp-mod-pppol2tp`).
3. **Определение архитектуры**: Детектирует CPU роутера (MIPSLE, ARM, ARM64 или standard Linux).
4. **Скачивание бинарника**: Автоматически скачает нужный скомпилированный файл веб-менеджера напрямую с GitHub в `/opt/usr/bin/l2tp-web`.
5. **Настройка логов**: Сконфигурирует strongSwan для записи подробных логов в `/opt/var/log/swanctl.log`.
6. **Автозапуск**: Создаст init-скрипт `/opt/etc/init.d/S99l2tp-web` для автозапуска веб-менеджера при старте роутера.
7. **Конфигурация**: Создаст пустой файл конфигурации `/opt/etc/l2tp_vpn_config.json` (если он еще не существовал).
8. **Запуск**: Запустит веб-интерфейс менеджера.

### Ручная установка (офлайн-метод)

Если у вашего роутера временно отсутствует интернет или вы хотите установить файлы локально:

1. Скачайте файлы [l2tp-setup.sh](https://raw.githubusercontent.com/l-ptrol/L2TP-VPN-Web-Manager/main/l2tp-setup.sh) и подходящий бинарник для вашей архитектуры (например, [l2tp-web-mipsle](https://raw.githubusercontent.com/l-ptrol/L2TP-VPN-Web-Manager/main/l2tp-web-mipsle)).
2. Скопируйте файлы на роутер в папку `/tmp/` с помощью SCP:
   ```bash
   scp l2tp-web-mipsle root@192.168.1.1:/tmp/
   scp l2tp-setup.sh root@192.168.1.1:/tmp/
   ```
3. Подключитесь по SSH и запустите скрипт из папки `/tmp/`:
   ```bash
   ssh root@192.168.1.1
   cd /tmp && chmod +x l2tp-setup.sh && ./l2tp-setup.sh
   ```

## Использование

### Добавление VPN-профиля

1. Нажмите **«Добавить профиль»** в веб-интерфейсе
2. Заполните поля:
   - **Название** — произвольное имя профиля (например, `Germany L2TP`)
   - **Сервер** — IP-адрес или домен VPN-сервера
   - **IPsec** — включите, если сервер требует L2TP/IPsec; укажите PSK (Pre-Shared Key)
   - **Логин / Пароль** — учётные данные VPN
   - **Тип аутентификации** — Auto, PAP, CHAP, MS-CHAP или MS-CHAP v2
   - **Внешний интерфейс** — WAN-интерфейс для подключения к VPN-серверу (можно выбрать несколько через запятую для failover)
   - **Порт SOCKS5** — порт локального SOCKS5-прокси (по умолчанию 1080)
   - **Автозапуск** — запускать этот профиль при старте роутера
3. Нажмите **«Сохранить»**

### Подключение

Нажмите тумблер (switch) на карточке профиля. Статус индикатора:

| Цвет | Состояние |
|------|-----------|
| Серый | Отключено |
| Жёлтый (мигает) | Подключение |
| Зелёный | Подключено |

### Использование SOCKS5-прокси

После успешного подключения туннеля запускается локальный SOCKS5-прокси на `127.0.0.1:<port>`. По соображениям безопасности прокси слушает только loopback-адрес роутера и недоступен для внешних сканеров или устройств в сети.

Чтобы использовать прокси на локальном компьютере или смартфоне в домашней сети, настройте перенаправление порта через SSH (замените `192.168.1.1` на LAN IP вашего роутера):

```bash
ssh -N -L 1080:127.0.0.1:1080 root@192.168.1.1
```

После этого в настройках браузера (например, через расширение SwitchyOmega) или в системе укажите параметры прокси:

```
Протокол: SOCKS5
Адрес: 127.0.0.1
Порт: 1080 (или указанный в профиле)
```

Прокси перенаправляет весь проходящий трафик через VPN-интерфейс, принудительно привязываясь к нему на уровне сокета, при этом остальной трафик роутера и других клиентов домашней сети продолжает идти напрямую через вашего провайдера.

### SSH-управление

Помимо веб-интерфейса, туннелем можно управлять через SSH:

```bash
# Статус туннеля:
ptrol-l2tp status

# Запуск:
ptrol-l2tp start

# Остановка:
ptrol-l2tp stop

# Перезапуск:
ptrol-l2tp restart
```

### Мониторинг логов

Веб-интерфейс отображает агрегированные логи из всех компонентов:
- **[Система]** — syslog (logread) / Keenetic CLI логи
- **[Менеджер]** — Go-бэкенд (запуск/остановка, мониторинг, ошибки)
- **[IPsec]** — swanctl.log (IPsec-переговоры, SA)
- **[L2TP-PPP]** — ppp-vpn.log (PPP-аутентификация, IP)
- **[L2TP]** — xl2tpd.log (L2TP-туннели, LNS)

## Конфигурация

Файл: `/opt/etc/l2tp_vpn_config.json` (на роутере) или `./l2tp_vpn_config.json` (на Windows)

```json
{
  "web_port": 8081,
  "profiles": [
    {
      "id": "profile_1700000000000000000",
      "name": "Germany L2TP",
      "server": "194.28.37.40",
      "use_ipsec": true,
      "psk": "your-pre-shared-key",
      "username": "vpnuser",
      "password": "vpnpass",
      "tun_num": 0,
      "out_interface": "ISP, UsbLte0",
      "auth_type": "auto",
      "enabled": false,
      "socks_port": 1080,
      "autostart": true
    }
  ],
  "active_id": ""
}
```

### Поля профиля

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | string | Название профиля |
| `server` | string | IP или домен VPN-сервера |
| `use_ipsec` | bool | Использовать IPsec (strongSwan) |
| `psk` | string | Pre-Shared Key для IPsec |
| `username` | string | Логин VPN |
| `password` | string | Пароль VPN |
| `tun_num` | int | Системный номер туннеля (влияет на имя ppp-интерфейса: `ppp<10+tun_num>`, по умолчанию 0) |
| `out_interface` | string | WAN-интерфейс(ы) через запятую с приоритетом |
| `auth_type` | string | Тип авторизации PPP: `auto`, `pap`, `chap`, `ms-chap`, `ms-chap-v2` |
| `socks_port` | int | Порт SOCKS5-прокси (по умолчанию 1080) |
| `autostart` | bool | Запуск этого подключения при старте службы менеджера |

## Как это работает

При запуске туннеля менеджер выполняет следующую последовательность:

1. **Остановка предыдущего туннеля** (если был активен)
2. **Определение исходящего интерфейса** — из списка `out_interface` выбирается первый активный с валидным IPv4
3. **Генерация конфигураций** — создаются файлы:
   - `/opt/etc/xl2tpd/xl2tpd.conf` — конфигурация xl2tpd
   - `/opt/etc/xl2tpd/xl2tpd-secrets` — пароль для L2TP
   - `/opt/etc/xl2tpd/options.l2tp` — PPP-опции
   - `/opt/etc/swanctl/swanctl.conf` — IPsec-конфигурация (если `use_ipsec`)
   - `/opt/etc/init.d/S99l2tp-vpn` — init-скрипт сервиса
4. **Запуск сервиса** — выполняется `/opt/etc/init.d/S99l2tp-vpn start`:
   - Добавляется маршрут к VPN-серверу через выбранный WAN-интерфейс
   - Запускается charon (IPsec), инициируется SA
   - Запускается xl2tpd с autodial
   - Ожидается появление ppp-интерфейса (до 30 секунд)
   - Настраивается iptables MASQUERADE
5. **Запуск SOCKS5-прокси** — после поднятия ppp-интерфейса запускается прокси, привязанный к VPN через `SO_BINDTODEVICE`
6. **Мониторинг** — watchdog каждые 5 секунд проверяет статус туннеля и активность интерфейса; при обрыве — автоматический перезапуск

Для сборки под все платформы:

```powershell
# Windows (PowerShell)
.\build.ps1
```

```bash

CGO_ENABLED=0 GOOS=linux GOARCH=mipsle GOMIPS=softfloat \

CGO_ENABLED=0 GOOS=linux GOARCH=arm64 \

CGO_ENABLED=0 GOOS=windows GOARCH=amd64 \
  go build -buildvcs=false -o l2tp-web.exe
```

### Параметры запуска (CLI Flags)

При запуске бинарного файла напрямую поддерживаются аргументы командной строки:
- `-port <число>` — Задать порт веб-панели (переопределяет и сохраняет порт в файле конфигурации).
- `-config <путь>` — Указать путь к файлу конфигурации (по умолчанию `/opt/etc/l2tp_vpn_config.json` на Linux и `./l2tp_vpn_config.json` на Windows).

Пример ручного запуска на порту `9000`:
```bash
/opt/usr/bin/l2tp-web -port 9000
```
